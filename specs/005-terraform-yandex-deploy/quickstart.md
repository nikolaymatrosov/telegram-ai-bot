# Quickstart: Terraform Yandex Cloud Deployment

**Feature**: 005-terraform-yandex-deploy

This guide walks through deploying the Telegram AI Bot to Yandex Cloud using Terraform.

---

## Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.0
- [Yandex Cloud CLI (`yc`)](https://yandex.cloud/en/docs/cli/quickstart) configured with a service account key
- Node.js 20 LTS + npm (for the local TypeScript build step)
- A Yandex Cloud folder with billing enabled
- A Telegram bot token (from [@BotFather](https://t.me/BotFather))
- An OpenAI API key

---

## Step 1: Configure Variables

```bash
cp tf/.tfvars.example tf/.tfvars
# Edit tf/.tfvars and fill in:
#   cloud_id       = "your-yandex-cloud-id"
#   folder_id      = "your-folder-id"
#   bot_token      = "123456789:ABCdef..."
#   openai_api_key = "sk-..."
```

The `WEBHOOK_SECRET` is auto-generated by Terraform — you do not set it manually.

---

## Step 2: Initialize and Apply

```bash
cd tf
terraform init
terraform plan          # review what will be created
terraform apply         # type "yes" to confirm
```

This will:
1. Build the TypeScript project (`npm ci && npm run build`)
2. Assemble and zip the function code with dependencies
3. Provision the YDB serverless database
4. Create service accounts and IAM roles
5. Deploy the webhook function and migration function
6. Run database migrations automatically
7. Print outputs including the function URL

---

## Step 3: Register the Telegram Webhook

After `terraform apply` completes, register the webhook using the project's built-in script:

```bash
# Get the generated webhook secret (redacted in terminal output)
WEBHOOK_SECRET=$(cd tf && terraform output -raw webhook_secret)

# Get the function URL
WEBHOOK_URL=$(cd tf && terraform output -raw webhook_function_url)

# Register with Telegram
BOT_TOKEN="<your-bot-token>" \
WEBHOOK_URL="$WEBHOOK_URL" \
WEBHOOK_SECRET="$WEBHOOK_SECRET" \
npx tsx src/webhook/scripts/set-webhook.ts
```

Or use the one-liner from the Terraform output:
```bash
cd tf && terraform output -raw set_webhook_command | bash
```

---

## Step 4: Verify

Send `/start` to your bot in Telegram. You should receive a response.

To verify the deployment manually:
```bash
cd tf
terraform output webhook_function_url
```

Then make a test request or check YCF function logs in the Yandex Cloud console.

---

## Updating the Bot

After making code changes:

```bash
cd tf
terraform apply
```

Terraform detects changed source file hashes, rebuilds the ZIP, updates the function, and re-runs migrations (idempotent — safe on every deploy).

---

## Teardown

```bash
# First, remove the Telegram webhook
BOT_TOKEN="<your-bot-token>" npx tsx src/webhook/scripts/delete-webhook.ts

# Then destroy all cloud resources
cd tf
terraform destroy
```

## Environment Variables Reference

The deployed function receives these environment variables (set by Terraform):

| Variable | Source |
|----------|--------|
| `BOT_TOKEN` | `var.bot_token` from `.tfvars` |
| `OPENAI_API_KEY` | `var.openai_api_key` from `.tfvars` |
| `YDB_ENDPOINT` | YDB resource output (auto) |
| `YDB_DATABASE` | YDB resource output (auto) |
| `WEBHOOK_SECRET` | Auto-generated; read with `terraform output -raw webhook_secret` |

`BOT_INFO` is not set — the function fetches it from Telegram on first cold start.

## Troubleshooting

**`yc` command not found during `terraform apply`**
Install the Yandex Cloud CLI and run `yc init` to authenticate before applying.

**Migrations fail (`null_resource.run_migrations` error)**
Check YCF function logs for `telegram-bot-migrate` in the Yandex Cloud console. Common cause: IAM propagation delay. Run `terraform apply` again — the migration function is idempotent.

**Function not responding to Telegram**
1. Check `terraform output webhook_function_url` — confirm the URL is correct
2. Verify the webhook is registered: `curl https://api.telegram.org/bot<TOKEN>/getWebhookInfo`
3. Check YCF function logs for `telegram-bot-webhook`

**`BOT_TOKEN` not accepted**
Ensure the value in `.tfvars` does not have quotes around it. Correct format: `bot_token = "123456:ABCdef"` (with quotes as part of HCL string syntax).
